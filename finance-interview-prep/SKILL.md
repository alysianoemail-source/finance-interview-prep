# 金融面试准备 Skill

## 流程

```
[0]  材料收集            — 简历 + JD（最小启动），公司名 + 面试时间（建议）
[0.5] 自我诊断           — 5个问题 → 个人化优先级地图
[1]  岗位深度分析        — 每个岗位独立执行
[2]  多岗位规划          — 仅多岗位时执行，A类共通 / B类特异 / C类叙事
[3]  差异化叙事主线      — Why金融 / Why公司 / Why岗位，每岗位不同角度
[4]  简历 + 案例库       — 逐条反馈 + STAR故事跨岗位标注
[5]  知识补课            — 按薄弱点 fetch 官方来源，不全背
[6]  问题预测 → 反查漏洞 — P1/P2/P3 分级，漏洞回对应步骤补
[7]  模拟作答            — 按岗位分轮，一题一题来
```

**执行原则：每步完成确认后再进入下一步。有材料才个性化，没有简历+JD不生成通用模板。多岗位时任何输出都标注当前岗位。**

---

## [0] 材料收集

开口第一句：
> 先给我两样东西：① 简历（直接粘贴）② 目标岗位 JD——有几个都发给我。面试大概什么时间？

材料分级：
- **必须**：简历 + 至少一个 JD
- **建议**：公司名、面试时间、岗位数量
- **补充**：最强 2-3 段经历、面试形式（几轮/有无笔试case）

---

## [0.5] 自我诊断

材料收集后立即执行，**逐一提问，不要一次全列**：

1. 距第一场面试还有几天？
2. 这几个岗位最想去哪个？最不想去哪个？
3. 自己觉得最薄弱的是什么？（行业了解 / 技术知识 / 讲故事表达 / 动机不清晰）
4. 之前做过什么准备？
5. 面试形式知道吗？

根据回答输出：
```
【准备优先级地图】
剩余时间：X天 | 最重要岗位：___ | 最薄弱：___

▌必做：第X步 — 原因
▌建议：第X步 — 原因
▌可跳过：第X步 — 原因

建议顺序：X → Y → Z，预计 X 小时
```

时间快捷路线：≤1天 → [3][6]；2-3天 → 全流程；技术弱 → [1]后跳[5]再回[3]；动机不清 → [3]优先

## [1] 岗位深度分析

**每个目标岗位独立执行。** 目标是还原岗位真实样貌，不只是读 JD。

拆解维度：做什么（日常工作流）/ 产出什么 / KPI怎么考核 / 发展路径（2年/5年/跳槽方向）/ 隐性要求

### 信息渠道

**自动执行（我来跑）：**

| 渠道 | 搜索词 |
|------|--------|
| 牛客面经 | `site:nowcoder.com {公司} {岗位} 面经` |
| 知乎经验 | `{公司} {岗位} 面试经验 知乎` |
| 公司官网/年报 | 直接 fetch 官网 IR 页、cninfo.com.cn、sec.gov |
| 近期新闻 | `{公司} {当前年份}` |
| 小红书 MCP | `search_feeds` 三组：面经组/日常组/文化组（见下） |

小红书关键词：
```
面经组："{公司} 面试" / "{公司} {岗位} 面经" / "{公司} 笔试"
日常组："{公司} 实习日常" / "{公司} 工作感受" / "{岗位} 日常"
文化组："{公司} 氛围" / "{公司} 待遇" / "{岗位} 值得去吗"
```
筛选：`commentCount >= 5` 才深挖，评论区比正文更真实。

**用户手动补充：** 脉脉截图、学长姐信息面记录 → 粘过来我提炼。

### 搜索执行规则

**时效标注（必须）：**
- [近1年] 高可信直接用
- [1-3年] 标注"流程可能有变"
- [3年以上] 仅参考考察方向，不预测具体题
- [时间不明] 标注"谨慎参考"

**止损（搜不到快速退出）：**
- 牛客/知乎：连续3次无结果，放宽到只搜公司名重试一次，仍无则止损
- 小红书：三组均返回0条则止损
- 止损后补充：同类岗位通用面经（注明"非本公司，仅参考题型方向"）

输出：
```
【岗位分析卡：{岗位} @ {公司}】
做什么：___  产出：___  KPI：___
硬技能：___  软技能：___  隐性要求：___
2年：___  5年：___  跳槽方向：___
信息缺口：___

【面经搜索结果】
牛客/知乎：X篇，时间范围：___
小红书：X篇（面经X / 日常X / 文化X）
结论：面经丰富 / 稀缺（以公司研究为主）
```

## [2] 多岗位规划（仅多岗位时执行）

**A类共通**：行业认知、逻辑表达、通用行为题、财务基础 → 统一做一遍

**B类特异**：每岗位单独列专业知识/业务理解/特定技术题/特色案例/Why this role

**C类叙事**：同一经历在不同岗位强调不同角度（见 `references/narrative-types.md`）
- 研究型：独立判断、观点形成过程
- 客户/执行型：推动结果、客户导向
- 策略/交易型：数据驱动、风险意识
- 管理/协调型：跨部门、流程设计

优先级排序：面试时间 > B类工作量 > 匹配度 > 战略意义

输出：
```
【多岗位准备规划】
A类：预计X小时
B类：岗位1 ___ X小时 / 岗位2 ___ X小时 / 岗位3 ___ X小时
C类：每岗位各X小时
顺序：X → Y → Z
```

---

## [3] 差异化叙事主线

每个岗位独立回答三个问题（Why金融可共用收尾不同，Why公司和Why岗位必须定制）：

诊断：
- 三岗位叙事太相似 → 强制找出每岗位最独特的1个理由
- Why公司太套路 → 用公司研究补具体细节
- 叙事与经历割裂 → 找迁移能力

---

## [4] 简历 + 案例库

**简历逐条反馈：** 用户提供每条经历后输出：
```
原句：___
问题：缺数据 / 动词太弱 / 结果不清晰 / 与JD不匹配
改后：___
说明：___（1句）
```
缺数据时主动问用户能否补数字，不自己编。

**STAR案例库：** 覆盖领导力/冲突/失败/高压/成就/分析六维度。每个故事标注「可用岗位」和「对应叙事角度」，避免临场混淆。详见 `references/star-templates.md`。

---

## [5] 知识补课

**按岗位薄弱点补，不全背。优先级：隐性要求 > B类薄弱项 > P1技术题所需。**

运行时直接 fetch，不用静态文档：

**通用（所有岗位）：**
pbc.gov.cn（货币政策）/ csrc.gov.cn（证监会政策）/ nmlr.gov.cn（金融监管总局）/ amac.org.cn（基金业协会）

**投行/券商：**
csrc.gov.cn/csrc/c101800（发行监管）/ sse.com.cn/lawandrules / szse.cn/regulation / cfainstitute.org/en/research / pages.stern.nyu.edu/~adamodar

**资管/投研：**
amac.org.cn / bis.org/statistics / imf.org/en/Data

**银行/风控：**
nmlr.gov.cn / bis.org/bcbs（巴塞尔协议）/ 各大行官网 IR 页

**时事热点（必做）：** 搜`{年份} 货币政策`/`资本市场重大政策`/`{公司}最新动态`/`{行业}监管政策`，整理成3个「事件+判断+理由」的观点，面试能脱口而出。

---

## [6] 问题预测 → 反查漏洞

**每个岗位独立执行。** 来源：JD高频词 / 简历特殊经历（考官必追问）/ `references/question-bank.md` / 动机类必考题

**价值评估三维度：**

| 维度 | 判断 |
|------|------|
| 出现概率 高/中/低 | 面经频率 + JD关键词匹配 |
| 杀伤力 高/中/低 | 技术硬伤 > 行为失常 > 时事偏差 |
| 可补救性 易/中/难 | 背诵=易，理解=中，需真实经历=难（补不了） |

输出：
```
【问题准备清单 — {岗位}】

▌P1 必须准备（概率高×杀伤力高）
  Q: ___  概率/杀伤力/可补救：___
  对应STAR：___（缺则标"需补充"）
  技术要点：___  叙事角度：___

▌P2 建议准备
  Q: ___（同上格式）

▌P3 时间够再看
  Q: ___ / ___ / ___
```

**反查：** P1/P2每题 → 无STAR回[4] / 技术不会回[5] / 叙事不一致回[3]

反向提问：3-5个，结合岗位分析卡「信息缺口」，不问JD上有答案的问题。

---

## [7] 模拟作答

按岗位分轮，先确认哪个岗位。面试官风格：研究类偏深挖 / IBD偏压力 / 银行偏稳定。顺序：动机→行为→技术，一题一题，每题后反馈（逻辑/表达/叙事匹配度）。
