# STAR 故事库模板

## 六个必备维度 + 示例框架

---

### 1. 领导力 / 影响力

**典型问题：**
- "在没有直接权力的情况下，你如何推动一个结果？"
- "描述一次你带领团队完成困难任务的经历"

**写作框架：**
```
情境：[项目/团队背景，你的正式角色]
任务：[需要推动的具体目标，以及你面对的阻力]
行动：
  - 如何建立信任/说服他人
  - 如何分配资源或协调分歧
  - 关键决策点你做了什么
结果：[量化成果] + [对团队/组织的影响]
```

**金融场景示例结构：**
> 在实习期间主导了一份行业研究报告，需要协调三个部门提供数据。通过[具体行动]，最终报告被用于[具体用途]，节省了[X]小时的重复工作。

---

### 2. 冲突 / 分歧处理

**典型问题：**
- "你和上级/同事意见不一致时怎么办？"
- "描述一次你需要说服别人改变想法的经历"

**写作框架：**
```
情境：[分歧的背景，双方立场]
任务：[需要解决的实质问题]
行动：
  - 如何倾听对方观点
  - 用什么方式表达不同意见（数据/逻辑/第三方视角）
  - 如何找到共识或做出妥协
结果：[最终决策] + [关系结果（合作关系是否改善）]
```

**注意：** 不要让自己"完全正确"——好的故事展示你能从分歧中学习。

---

### 3. 失败 / 挫折

**典型问题：**
- "讲一个你失败的经历"
- "你最大的弱点是什么，有什么例子？"

**写作框架：**
```
情境：[失败事件的背景]
任务：[当时的目标]
行动：[你做了什么，哪里出了问题]
结果：
  - 直接结果（失败的具体表现）
  - 你从中学到什么（要具体，不能只说"我学到要更努力"）
  - 之后如何改变行为（有没有后续验证）
```

**金融语境：** 可以是一次错误的投资判断、模型假设失误、客户沟通失败。

---

### 4. 高压 / 截止日期

**典型问题：**
- "你如何在极端压力下工作？"
- "讲一次你同时处理多个紧急任务的经历"

**写作框架：**
```
情境：[压力来源，具体的时间/资源限制]
任务：[需要同时完成的多个目标]
行动：
  - 如何优先排序（明确说出你的标准）
  - 如何管理利益相关方预期
  - 遇到新问题如何应对
结果：[完成情况] + [质量是否达标]
```

---

### 5. 成就 / 超预期

**典型问题：**
- "你最自豪的一个成就是什么？"
- "你做过最有影响力的事情是什么？"

**写作框架：**
```
情境：[背景，为什么这件事有难度]
任务：[初始目标]
行动：[你做了什么超出预期的事]
结果：
  - 量化成果（数字越具体越好）
  - 对更大范围的影响
  - 为什么你认为这是你最好的成就
```

**量化建议：** 金融行业非常看重数字。尽量包含：规模（管理资产/交易金额）、增幅（%）、时间（节省多少）、排名（在团队/全行中的位置）。

---

### 6. 分析 / 解决问题

**典型问题：**
- "描述一次你解决复杂问题的经历"
- "你如何做一个困难的决策？"

**写作框架：**
```
情境：[问题的背景和复杂性]
任务：[需要回答的核心问题]
行动：
  - 如何拆解问题（结构化思维）
  - 用什么分析方法或数据
  - 如何处理不确定性或信息不足
结果：[结论/决策] + [验证方式]
```

---

## STAR 故事质量检查清单

写完每个故事后，对照检查：

- [ ] 有没有具体数字？（金额、比例、时间、排名）
- [ ] "行动"部分是你的主导，而不是团队？（用"我"而不是"我们"）
- [ ] 结果是否真的回答了问题？（不是泛泛说"项目成功了"）
- [ ] 故事是否与叙事主线一致？（体现你应聘这个岗位的核心能力）
- [ ] 时长控制在 2 分钟以内？（金融面试节奏快）
- [ ] 有没有可以延伸的细节？（面试官可能追问）

---

## 一个故事，多种用法

同一段经历可以根据问题侧重点不同，调整强调的部分：

| 同一段经历 | 回答"领导力"问题 | 回答"分析"问题 | 回答"压力"问题 |
|-----------|---------------|--------------|--------------|
| 做了一个复杂的行业研究 | 强调如何协调多方资源 | 强调分析框架和方法论 | 强调在紧张时间内完成 |

提前想好每段经历的"多角度用法"，面试时可以灵活调用。
